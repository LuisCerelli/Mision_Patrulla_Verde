# Misión Patrulla Verde

Interactive web-based Learning Object developed to support environmental education with primary school students through storytelling, comprehension activities, decision-making, community action, and a final environmental commitment.

## Description

**Mission Green Patrol** is an educational and interactive web application inspired by the story of a Green Patrol in the neighborhood.

The experience follows Sebastián, the main character of the story, on a mission to identify an environmental problem in the Villa Mar neighborhood, make decisions, organize a Green Patrol with other children, create environmental messages, and write a personal commitment to care for the environment.

The proposal encourages students to understand that environmental problems can be addressed through small, concrete actions carried out collectively in their closest surroundings: home, school, and neighborhood.

## Educational Objective

The objective of this Learning Object is for students to:

- identify a nearby environmental problem;
- recognize concrete actions to care for the neighborhood and the environment;
- understand the importance of teamwork;
- create environmental awareness messages;
- write a personal environmental commitment;
- generate a final evidence document of their learning path.

## Origin of the Proposal

This Learning Object was developed from an idea that emerged in a context of child participation, while preserving the identity of the child and other participants.

The intention is to recognize the value of children’s ideas and show that their contributions can be heard, developed, and transformed into real educational resources.

## Learning Path Structure

The Learning Object is organized into 10 screens:

1. Mission welcome.
2. Student registration.
3. Story introduction.
4. Environmental problem observation.
5. Decision-making.
6. Building the Green Patrol.
7. Creating environmental messages.
8. Mission result.
9. Final commitment.
10. Downloadable final evidence.

## Student Data

In this first version, student data is used only inside the browser to personalize the experience and generate the final evidence.

No student data is sent to external servers.
